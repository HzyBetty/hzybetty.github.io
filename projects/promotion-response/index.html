The problem

GitHub's documentation is vast and fragmented. Users spend unnecessary time navigating support articles for routine workflows such as enabling 2FA, managing billing, or troubleshooting repositories.

This project consolidates those workflows into a single conversational assistant capable of grounded documentation Q&A, ticket operations, CRM workflows, and safety guardrails — all backed by persistent storage and retrieval transparency.

RAG knowledge Q&A

User questions are routed to a vector retriever backed by scraped GitHub documentation. Retrieved chunks are passed into Llama 3.1 through Groq's API, and generated responses include inline citations to source URLs.

Cited answer with retrieval transparency
Query: "How do I set up two-factor authentication on GitHub?"

Architecture

Every incoming message passes through an intent router that combines regex pattern matching with LLM fallback classification. Requests are dispatched to isolated capability handlers while sharing a common persistence and memory layer.
